# Blognote Media Players

Простой аудиоплеер для блокнота, созданный на основе плеера Youvi.

## Возможности

### Аудиоплеер
- Воспроизведение/пауза
- Перемотка ±10 секунд
- Регулировка громкости
- Визуальный прогресс-бар с ползунком
- Красивый градиентный дизайн
- **Ограничение**: максимальный размер аудио - 50MB

## Использование в редакторе

### Добавление аудио
1. Нажмите кнопку "Аудио" в панели инструментов редактора
2. Выберите аудио файл (до 50MB)
3. В текст будет вставлен код: `[audio:Название](путь/к/файлу)`

## Технические детали

### Структура файлов
```
blognote/
  player/
    video-player.css    - Стили видеоплеера
    video-player.js     - Логика видеоплеера
    audio-player.css    - Стили аудиоплеера
    audio-player.js     - Логика аудиоплеера
```
Видео**: `[video](posts/post_xxx/video_xxx.mp4)`
- **Аудио**: `[audio:Название трека](posts/post_xxx/audio_xxx.mp3)`
Автоинициализация
Плееры автоматически инициализируются при загрузке страницы для всех элементов с атрибутами:
- `data-blognote-video` - для видео
- `data-blognote-audio` - для аудио
 автоматически инициализируется при загрузке страницы для всех элементов с атрибутом:

#### Видео
- MP4 (H.264)
- Web
- OGG
- M4A

## Интеграция

Плееры подключаются на странице `post.html`:

```html
<!-- CSS -->
<link rel="stylesheet" href="player/video-player.css">
<link rel="stylesheet" href="player/audio-player.css">
 подключается на странице `post.html`:

```html
<!-- CSS -->
<link rel="stylesheet" href="player/audio-player.css">

<!-- JavaScript -->
<script src="player/audio-player.js"></script>
```

## APIst player = new BlognoteAudioPlayer(containerElement, audioUrl, title);

// Методы
player.togglePlay()           // Переключить play/pause
player.seek(event)            // Перемотка
player.toggleMute()           // Включить/выключить звук
player.setVolume(value)       // Установить громкость (0-100)
```

## Стилизация

Плееры поддерживают темную тему блокнота через CSS-переменные и классы.

### Основные цвета
- Акцентный цвет: `#ff69b4` (розовый)
- Видеоплеер фон: черный с белыми элементами управления
- Аудиоплеер фон: градиент `#667eea` → `#764ba2`

## Примечания

- Все медиафайлы сохраняются локально через File System Access API
- Файлы хранятся в папке `posts/{postId}/`
- Имена файлов генерируются автоматически с timestamp
- При превышении лимита размера пользователь получит уведомление
